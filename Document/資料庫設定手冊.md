🌻## web/admin.html 要求輸入信箱與密碼的用途

用途：用 Supabase Auth（Email/Password）登入，取得 authenticated 身分（session）。
為什麼需要：你在 supabase/schema.sql 啟用了 RLS，並且 policy 設定為 authenticated 才能對管理資料表做寫入/讀取；未登入（anon）會被擋（或只允許公開讀取的表）。

🌻## 在 Supabase（LoveIsABitMessy-DB）建立「管理人」身分流程（建議做法） 以下用 Supabase Dashboard 直接建立管理者帳號（之後 admin.html 只要登入一次，就會因 session 持久化而「下次自動登入」）。

1) 開啟 Email/Password 登入
進入 Supabase 專案 LoveIsABitMessy-DB
左側 Authentication → Providers
確認 Email 已啟用（Email/Password）
2) 建立管理者帳號（用 Dashboard 建）
左側 Authentication → Users
點 Add user
輸入你的管理者 Email 與 Password
建立後，通常會出現在 users 清單中（是否需要 email confirm 取決於你 Auth 設定；用 Add user 建立通常可直接可用）
3) 用這個帳號登入後台
在 web/js/supabaseClient.js 填好 SUPABASE_URL 與 SUPABASE_KEY（只用 anon/publishable key）
打開 docs/admin.html（GitHub Pages）或本機 web/admin.html
輸入 Email/Password 按 登入
登入成功後就能管理資料表（符合 RLS policy）

最小範例 SQL（建立一個管理者清單表，可選用）
如果你想額外有一張「管理者 email 白名單」表（未來可用來加更嚴格的 RLS），可以先建立：

create table if not exists public.admin_allowlist (
  id uuid primary key default gen_random_uuid(),
  email text not null unique,
  note text,
  created_at timestamptz not null default now()
);
alter table public.admin_allowlist enable row level security;
-- 只有登入者可讀（也可改成只有特定 email 可讀，後續再加）
create policy p_admin_allowlist_auth_read
on public.admin_allowlist
for select
to authenticated
