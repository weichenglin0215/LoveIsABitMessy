# 角色設計規範

## 愛情人格特質評量表 (LPAS)題目設計規範
每一段期間(Period)都分成四個Dim
1. 靠近與表達 DIM，重點在於方式，例如：付諸行動、僅限言語、直接外約、巧合碰面、書信往來、社交通訊、運動健身、製造肢體接觸...。
2. 受傷消化 DIM，重點在於是否如何下決定？例如：直接表達？或是委屈隱忍？愛了就不在乎？越痛越愛？先提分手？只能我負人？一定要被傷透了還在替對方說好話？
3. 告別疏遠 DIM，重點在於能否下定決心？果斷分手、委曲求全、藕斷絲連、還想嘗試、尋求共同友人協助、找下一段戀情、消失療傷...
4. 關係節奏 DIM，評量重點在於時間快慢。

5. 絕對避免使用多重否定字問法,會讓受測者無法直覺回答。例如:"曖昧沒有結果,我寧可早一點結束,也不想在沒希望的地方耗著。" -> "長期處於曖昧,我寧可早一點結束,離開令人失望的對象。" 

6. 盡量不要使用否定字問法,會讓受測者無法直覺回答。例如:刪除"不"字,"另一半說了讓我很不舒服的話,我通常會告訴他我受傷了。" -> "另一半說出令我難過的話,我通常會告訴他,我受傷了。"

7. 使用中文文法,勿使用英文或日文文法。例如:奇怪文法的"熱戀期,「我喜歡你」或「我想你」這種話我說得很自然。" -> 合乎中文文法的"熱戀期,我很自然地說出「我喜歡你」或「我想你」。"

8. 使用中文文法結構(時間) + 主詞 + (否定) + (副詞/方式) + (把+受詞) + 動詞 + (助詞) + (形容詞) + (名詞受詞)

9. 一定要以繁體中文回答

請以上述規範，設計60個題目。

## 愛情人格特質評量表 (LPAS) 轉換標準
每位角色卡應由 LPAS 結果轉換而來，必須包含以下基本欄位以確保 AI 角色擁有一致性與立體感：

- `id`: 小寫英文與底線 (e.g., `lin_xiaoqing`)
- `name`: 角色全名(中文繁體)
- `age`: 年齡
- `position`: 職業或身份
- `appearance`: 外貌特徵
- `personality`: 核心個性與特質
- `speech_style`: 說話風格與口頭禪
- `relationship`: 與主角的關係
- `habits`: 習慣，應包含 3 項具體日常習慣陣列
- `image_prompt`: 給予 ComfyUI 繪圖使用的完整特徵 prompt (建議英文)

所有角色卡必須存為 JSON 集中放置在 `characters/` 內。
